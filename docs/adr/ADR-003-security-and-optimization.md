## CRÍTICO — Debe resolverse antes de `git push` a producción

### C1: SECRET_KEY con valor por defecto inseguro

**Archivo**: `backend/app/core/config.py:17`

```python
SECRET_KEY: str = "my_super_secret_for_jwt"  # ← CRÍTICO
```

Cualquiera que lea el código puede forjar JWTs válidos. En producción este valor
es el único mecanismo que protege todas las sesiones de la app.

**Fix — `backend/.env` (producción):**
```bash
# Generar en el VPS:
python3 -c "import secrets; print(secrets.token_hex(64))"
# Pegar el resultado en:
SECRET_KEY=<resultado_de_64_chars_random>
```

### C2: Endpoint de upload sin autenticación

**Archivo**: `backend/app/api/routers/uploads.py:15`

```python
@router.post("/image")
async def upload_image(file: UploadFile = File(...)):
    # ← Sin ningún Depends de auth
```

Cualquier persona puede subir archivos al VPS sin estar logueada.

**Fix:**
```python
from app.api.deps_security import get_current_user
from app.models.user import User
from fastapi import Depends

@router.post("/image")
async def upload_image(
    file: UploadFile = File(...),
    current_user: User = Depends(get_current_user),  # ← agregar esto
):
```

### C3: Token JWT en cookie no-HttpOnly (XSS risk)

**Archivo**: `frontend/src/lib/api.ts:2` y `components/auth-provider.tsx`

```typescript
import Cookies from "js-cookie";
const token = Cookies.get("access_token"); // js-cookie = accesible desde JS = XSS risk
```

`js-cookie` escribe cookies accesibles desde JavaScript. Si hay cualquier XSS
(incluso de una dependencia npm comprometida), el atacante roba el JWT.

**Fix — usar httpOnly cookies via un Route Handler de Next.js:**

```typescript
// frontend/src/app/api/auth/login/route.ts  (nuevo archivo)
import { NextRequest, NextResponse } from "next/server";

export async function POST(req: NextRequest) {
  const body = await req.json();

  const apiRes = await fetch(
    `${process.env.INTERNAL_API_URL}/api/v1/auth/login`,
    {
      method: "POST",
      headers: { "Content-Type": "application/x-www-form-urlencoded" },
      body: new URLSearchParams({ username: body.username, password: body.password }),
    }
  );

  if (!apiRes.ok) {
    const err = await apiRes.json();
    return NextResponse.json(err, { status: apiRes.status });
  }

  const data = await apiRes.json();
  const res = NextResponse.json({ rol: data.rol, nombre: data.nombre });

  res.cookies.set("access_token", data.access_token, {
    httpOnly: true,
    secure: true,
    sameSite: "lax",
    maxAge: 60 * 60 * 24 * 7, // 7 días
    path: "/",
  });

  return res;
}
```

> **Nota**: Este fix cambia la arquitectura de auth y requiere ajustar
> `auth-provider.tsx`. Se puede diferir al sprint post-deploy si el timeline
> es ajustado, pero debe hacerse antes del lanzamiento público real.
> En la fase inmediata, mitigar con un CSP header estricto en nginx (ver nginx config ADR-002).

**Variable de entorno adicional** (`frontend/.env.production`):
```env
# URL interna (server-side), nunca expuesta al browser
INTERNAL_API_URL=http://127.0.0.1:8000
```

## MEDIO — Resolver en la misma sesión

### M1: `next.config.ts` permite imágenes de cualquier dominio HTTPS

**Archivo**: `frontend/next.config.ts:12`

```typescript
{ protocol: 'https', hostname: '**' }  // ← cualquier dominio
```

Permite que `<Image>` de Next.js haga proxy de imágenes de cualquier servidor
externo, lo que puede ser abusado para SSRF o simplemente es innecesario.

**Fix:**
```typescript
// frontend/next.config.ts
images: {
  remotePatterns: [
    {
      protocol: 'https',
      hostname: 'alexis.iwebtecnology.com',
      pathname: '/media/images/**',
    },
    {
      protocol: 'http',
      hostname: 'localhost',  // solo dev
    },
  ],
},
```

## OPTIMIZACIONES

### O1: Video del Hero — comprimir y agregar `poster`

**Archivo**: `frontend/src/components/home/HeroSection.tsx:14`

```html
<video src="/resources/hero.mp4" autoPlay loop muted playsInline ...>
```

El video se carga sin poster (pantalla negra mientras carga) y probablemente
pesa varios MB sin comprimir.

**Acciones:**
```bash
# En el VPS, instalar ffmpeg y comprimir:
ffmpeg -i hero.mp4 \
  -vf "scale=1920:-2" \
  -c:v libx264 -crf 28 -preset slow \
  -an \
  -movflags +faststart \
  hero-optimized.mp4

# Extraer frame 0 como poster:
ffmpeg -i hero.mp4 -vframes 1 -q:v 2 hero-poster.jpg
```

```tsx
<video
  src="/resources/hero-optimized.mp4"
  poster="/resources/hero-poster.jpg"  // ← evita pantalla negra
  autoPlay loop muted playsInline
  preload="none"                        // ← no bloquea el LCP
/>
```

### O2: Rate limiting en el endpoint de login

**Archivo**: `backend/app/api/routers/auth.py:16`

Sin rate limiting, el endpoint `/auth/login` es vulnerable a fuerza bruta.

**Fix — instalar `slowapi`:**
```bash
pip install slowapi
```

```python
# backend/app/main.py
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address
from slowapi.errors import RateLimitExceeded

limiter = Limiter(key_func=get_remote_address)
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)
```

```python
# backend/app/api/routers/auth.py
from slowapi import Limiter
from slowapi.util import get_remote_address
from fastapi import Request

limiter = Limiter(key_func=get_remote_address)

@router.post("/login")
@limiter.limit("10/minute")
def login_for_access_token(request: Request, ...):
    ...
```

### O3: Migrar de SQLite a MySQL en producción

**Archivo**: `backend/app/core/config.py:16`

SQLite no soporta concurrencia real. Con múltiples workers de uvicorn, se
producirán errores de `database is locked`.

**`backend/.env` (producción):**
```env
DATABASE_URL=mysql+pymysql://alexisevt_user:PASSWORD@localhost/alexisevt_db
```

**Setup en VPS:**
```bash
# Instalar dependencia
pip install pymysql

# Crear base de datos
mysql -u root -p <<EOF
CREATE DATABASE alexisevt_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'alexisevt_user'@'localhost' IDENTIFIED BY 'PASSWORD_SEGURO';
GRANT ALL PRIVILEGES ON alexisevt_db.* TO 'alexisevt_user'@'localhost';
FLUSH PRIVILEGES;
EOF

# Correr migraciones
cd /home/iweb/alexisevt/project/backend
alembic upgrade head
```

app = FastAPI(
